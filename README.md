## О работе

Исследование посвящено снижению электропотребления системы вентиляции большого кинозала
на 135 мест (объект — СПб ГБУК «Петербург-кино», Ярославский пр., 55).
Базовая система работает с постоянным расходом воздуха круглые сутки независимо от заполнения зала,
что приводит к избыточным энергозатратам.

Для решения задачи построена расчётная модель в **OpenStudio / EnergyPlus** и проведено
почасовое энергетическое моделирование **шести сценариев** на горизонте одного года.
По результатам сравнения выбран оптимальный вариант, для которого разработана
схема автоматического управления с реализацией в среде **Node-RED**.

## Сценарии моделирования

| № | Сценарий | Краткое описание |
|---|----------|-----------------|
| 0 | Base | Базовый — постоянный расход, нет рекуперации |
| 1 | Recuperator | Роторный рекуператор КПД 85 % |
| 2 | Heat Pump | Тепловой насос «воздух–воздух» COP ≈ 2,5 |
| 3 | DCV | Управление по CO₂ (уставка 700 ppm, мин. расход 50 %) |
| 4 | DCV + Recuperator | DCV + роторный рекуператор |
| 5 | DCV + Heat Pump | DCV + тепловой насос |

**Оптимальный сценарий — DCV (сценарий 3):**
экономия электроэнергии **56,8 %** в год, срок окупаемости **0,7 лет**, NPV **1 254 тыс. руб.**

## Принцип работы (DCV-управление)

```
         CO₂-датчик (зал)
               │
               ▼
    ┌──────────────────────┐
    │  Контроллер (ПЛК)    │
    │  Уставка = 700 ppm   │
    │  ПИД-регулятор       │
    └──────────┬───────────┘
               │ сигнал 0–10 В
               ▼
    Частотный преобразователь
               │
               ▼
      Вентилятор приточный
    (расход 50–100 % от ном.)
```

1. CO₂-датчик замеряет концентрацию в зале каждые 30 с.
2. Если CO₂ < 700 ppm — расход снижается до минимума (50 % от номинала).
3. Если CO₂ > 700 ppm — ПИД-регулятор увеличивает скорость вентилятора.
4. При CO₂ > 1 000 ppm — принудительный выход на 100 % расход.
5. Алгоритм реализован в Node-RED (`automation/node_red_flow.json`).

## Ключевые результаты

| Показатель | DCV | DCV + Рекуп. | DCV + ТН |
|------------|-----|-------------|---------|
| Срок окупаемости, лет | **0,7** | 1,2 | 1,92 |
| NPV (10 лет), тыс. руб. | **1 254** | 1 375 | 1 298 |
| IRR | **144 %** | 83 % | 52 % |
| Экономия э/э в год | **56,8 %** | ~57 % | ~57 % |

Рекомендован сценарий **DCV** как оптимальный по совокупности критериев:
наименьший срок окупаемости, наивысший IRR, минимальная стоимость внедрения.

## Технический стек

- **OpenStudio 3.7 / EnergyPlus 23.2** — почасовое энергетическое моделирование
- **Python 3.10+** — постобработка результатов, генерация отчёта
- **Node-RED** — реализация алгоритма DCV-управления
- **ПЛК110-220.60.Р** — программируемый логический контроллер
- **Датчик CO₂ Вентс МС-620** — измерение концентрации углекислого газа

---

## Как импортировать Node-RED flow

1. Открыть Node-RED (`http://localhost:1880`).
2. Меню → **Import** → вставить содержимое `automation/node_red_flow.json`.
3. Нажать **Deploy**.

```bash
# Открыть базовую модель

# Открыть DCV-модель
openstudio app models/kinozal_DCV.osm
```
